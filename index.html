Project Overview

I built a serverless file sharing and real-time collaboration platform that allows users to securely upload files, share them with specific people, and comment instantly without refreshing the page.

The goal was to create a simpler, faster collaboration experience compared to heavy file-sharing tools.

Technology Stack

AWS S3, Lambda, DynamoDB, AppSync, Cognito, CloudWatch, Amplify, GitHub CI/CD, GraphQL

Product Decisions

  • Used serverless architecture to reduce cost and avoid managing servers.
  • Used AppSync because it supports real-time GraphQL subscriptions.
  • Used S3 presigned URLs to make uploads faster and reduce backend processing.
  • Used Cognito to secure access and support user-based file sharing.
  • Used CloudWatch to monitor errors and troubleshoot issues quickly.

Challenges I Solved

  • Fixed GraphQL subscription issues for real-time comments.
  • Improved the access control model so sharing was secure but still easy to use.
  • Used CloudWatch logs to investigate upload, permission, and Lambda errors.
  • Balanced technical complexity with a simple user experience.

Outcome

Delivered a working MVP from idea to deployment, showing ownership across product thinking, cloud architecture, debugging, and iteration.

What I Would Improve Next
